            I tried to find out info on the Runtime Error 10047, and it was explained at some websites. The first text is quite old (has something about MS-DOS), so it might not be the same. here is the text:

            10047 Console mode behavior problems with Value Procedure & Function.

            There have been changes introduced in Console Mode which mean

            that code which augments the SET VALUE procedure and the VALUE

            function will no longer work as it does in DOS Mode (in 3.1 and 3.1c).

            An example program, where the intention is to have the value

            of an entry item to appear to be "Y" for internal purposes

            (i.e. storing in variables via ENTRY_UPDATE, or any other

            accesses via the VALUE function) but to be displayed as "YES"

            on the screen: If you run this program under 3.1c DOS Mode and immediately press F1,

            the displayed value of item 1 (the second item) changes to "YES",

            which is the intention.

            then again, here is another explanation:

            Q126158 SMS Administrator Err Msg: Socket () failed, 10047

            and third explanation:

            After installing Rhapsody on my system I get the following message: Rhapsody cannot create socket (10047) Check for another copy of Rhapsody running. Annimation will be disabled!!! What do i do? Most likely you already have one instance of Rhapsody running and you're trying to invoke another one. You can have more than one instance of Rhapsody running, however, you will not be able to run animation

            so most propably its a socket error.
